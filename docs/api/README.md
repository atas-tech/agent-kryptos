# API Reference

BlindPass now ships a maintained OpenAPI snapshot for the stable hosted and local-development surfaces used by the dashboard, browser UI, and agent bootstrap flows.

- OpenAPI document: [openapi.yaml](../../docs/api/openapi.yaml)

## Coverage

The current spec covers:

- health and readiness endpoints
- hosted user auth, session refresh, and locale preference updates
- workspace and policy management
- agent enrollment and API-key token minting
- human-to-agent secret request and retrieval routes
- dashboard billing and analytics endpoints

It is a manual snapshot derived from the route registrations and request schemas in `packages/sps-server/src/routes`. Lower-level guest-intent and some advanced exchange-fulfillment paths are still better read from the source until the spec is expanded further.

## Auth model summary

- User-facing dashboard routes use bearer JWT auth.
- Hosted refresh uses `httpOnly` cookies on `/api/v2/auth/refresh` and `/api/v2/auth/logout`.
- Auth user payloads include `preferred_locale` so the dashboard can re-apply the stored language after authentication and refresh.
- Registration accepts optional `preferred_locale`, and `PATCH /api/v2/auth/locale` persists explicit language changes for the current user.
- Verification and password-reset email flows prefer stored `preferred_locale`; `Accept-Language` is only a fallback when no stored preference exists.
- Agent bootstrap uses `x-agent-api-key` on `POST /api/v2/agents/token`.
- Agent operational routes use bearer JWT auth minted by SPS.
- Browser submit/metadata routes use scoped HMAC query signatures generated by SPS.
